Workspace Config

How repos map to display names, colors, and behavior

~/.config/kittylauncher/config.yaml
# Global settings
repos_dir: ~/repos
scratch_dir: /tmp/kl-scratch
default_action: claude # or "shell"
# Workspace definitions
workspaces:
SliceWise: # matches ~/repos/SliceWise
name: "SliceWize"
color: "#ff6b6b"
remote: true
favourite: true
tgclaudebridge:
name: "Telegram Bridge"
color: "#0088cc"
remote: true
favourite: true
vinted-deals:
name: "Vinted"
color: "#09b1ba"
lom2:
name: "Legend of Mir"
color: "#c792ea"
favourite: true

What should be configurable?

A

Config file only

Edit YAML manually. TUI is read-only — it reads config on startup. Simple, predictable.

B

TUI can edit config

Toggle remote/favourite from the TUI, set names inline. Changes persist back to YAML. More convenient, more code.

C

Auto-discover + overrides

TUI scans ~/repos automatically. Config only needed for overrides (name, color, remote). Unconfigured repos show with dirname and default color.

I'd recommend C — auto-discover everything, config just for the special ones. What do you think?
